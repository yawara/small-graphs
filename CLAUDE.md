# CLAUDE.md

このリポジトリで作業する Claude / Codex / その他のエージェント向けの指示です。

## プロジェクトの目的

`small-graphs` は、degree-diameter problem に関係する小さい有限グラフの性質を、日本語の自然言語で整理するためのリポジトリです。

最終的には Lean による形式化を目指しますが、現段階では Markdown による数学的記述を主成果物とします。

## 作業時の原則

- 本文は日本語で書いてください。
- 数式、群、グラフの標準記法は通常の数学記法を使ってよいです。
- 数学的主張は、可能な限り定義、命題、証明スケッチ、補題候補に分けてください。
- 断言する命題と、まだ作業中の予想・観察は明確に分けてください。
- Lean 化を意識して、曖昧な言葉を避けてください。「たぶん」「明らか」だけで済ませず、Lean 化しやすい補題へ分解してください。
- 頂点集合、辺集合、隣接関係、写像、群の生成元はできるだけ明示的に書いてください。
- 既知事実を使う場合は、可能な限りどの部分が既知事実で、どの部分がこのリポジトリ内の確認かを分けてください。
- 既存の Markdown を大きく書き換えるときは、数学的意味を変えていないか注意してください。
- 誤りを見つけた場合は、遠慮なく修正し、なぜ誤りかを短く説明してください。

## Markdown メモの形式

新しいグラフのメモを追加する場合は、`notes/` 以下に置いてください。

ファイル名は英小文字とハイフンを使い、例として次のようにします。

```text
notes/mobius-ladder-m8.md
notes/petersen-graph.md
notes/cube-q3.md
```

各ノートはおおむね次の構成にしてください。

1. 対象グラフの定義
2. 基本性質
3. 距離・直径
4. 自己同型群
5. Lean 化のための補題候補
6. 未解決・未確認事項

## Lean 化時の方針

Lean ファイルを追加する段階では、まず自然言語ノート内の命題に対応する小さな補題を作ってください。

特に、次の順に進めるのが望ましいです。

1. 頂点型の定義
2. 隣接関係の定義
3. 単純グラフとしての構成
4. 次数の証明
5. 直径の証明
6. 自己同型の構成
7. 自己同型群の位数または同型型の証明

自然言語メモを書く段階でも、次の点を意識してください。

- 頂点集合は有限型として定義しやすい形にする。例: `ZMod 8`, `Fin 8`。
- 辺関係は決定可能な述語として書けるようにする。
- グラフ距離の証明は、各頂点からの距離層を列挙する形にすると形式化しやすい。
- 自己同型群の証明は、生成元を与える部分と、それで全てであることを示す部分に分ける。
- 位数計算で軌道安定化定理を使う場合は、その前提を明示する。

まだ Lean 化していない自然言語メモを、勝手に完全証明済みとして扱わないでください。

## 禁止事項・注意事項

- 検証されていない命題を「定理」として置かない。
- 記法 `D_8` など、文献により意味が揺れるものは必ず位数や表示を併記する。
- 外部の計算機実験を使った場合は、その計算が人間の証明なのか、単なる確認なのかを明記する。
