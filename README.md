# small-graphs

小さいグラフの性質を自然言語で記録し、あとから Codex や Lean を使って形式化していくためのリポジトリです。

主な対象は degree-diameter problem に関連する有限グラフです。最初は小さい例から始め、自然言語での観察、補題、証明スケッチ、反例、自己同型群の計算などを Markdown として蓄積します。

## 目的

このリポジトリの目的は、完成した Lean 証明を最初から書くことではなく、まず人間に読める数学的記述を整備することです。

自然言語メモでは、次の情報を明示することを重視します。

- グラフの定義
- 頂点集合と辺集合
- 次数、直径、正則性、連結性
- 自己同型群
- 証明したい命題
- Lean 化するときに必要になりそうな補題
- 未確認の主張、計算で確認した主張、人間が確認した主張の区別

## 記法

このリポジトリでは、特に断らない限りグラフは単純無向グラフとします。

頂点集合を `V`、辺集合を `E` と書きます。距離はグラフ距離です。直径は全ての頂点対の距離の最大値です。

二面体群の記法には混乱があるため、必要に応じて位数を明示します。たとえば正八角形の対称群は位数16の二面体群であり、ここでは必要なら

```text
D_8 = <r, s | r^8 = s^2 = 1, srs = r^{-1}>
```

のように表示します。

## Lean 化への方針

当面は自然言語の Markdown を正本とし、後から Lean ファイルに対応する命題を切り出します。

想定する流れは次の通りです。

1. Markdown でグラフの定義と命題を書く。
2. 命題を小さな補題に分解する。
3. 頂点集合を有限型として定義する。
4. 辺関係を決める。
5. 次数、直径、自己同型群などを Lean で証明する。

最初の対象は `M8` と `K_{3,3}` の1頂点三角形展開です。
