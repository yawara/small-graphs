# AGENTS.md

このリポジトリは、小さい有限グラフについての自然言語メモを、将来的に Lean 証明へ移行するための作業場です。

## 基本方針

- 自然言語の本文は日本語で書く。
- 数式、群、グラフの標準記法は通常の数学記法を使ってよい。
- 断言する命題と、まだ作業中の予想・観察を明確に分ける。
- 形式化前のメモであっても、頂点集合、辺集合、隣接関係はできるだけ明示的に書く。
- 「たぶん」「明らか」だけで済ませず、Lean 化しやすい補題へ分解する。
- 既知事実を使う場合は、可能な限りどの部分が既知事実で、どの部分がこのリポジトリ内の確認かを分ける。

## Markdown の書き方

各ノートはおおむね次の構成にする。

1. 対象グラフの定義
2. 基本性質
3. 距離・直径
4. 自己同型群
5. Lean 化のための補題候補
6. 未解決・未確認事項

## Lean 化を見越した注意

自然言語メモを書くときも、次の点を意識する。

- 頂点集合は有限型として定義しやすい形にする。例: `ZMod 8`, `Fin 8`。
- 辺関係は決定可能な述語として書けるようにする。
- グラフ距離の証明は、各頂点からの距離層を列挙する形にすると形式化しやすい。
- 自己同型群の証明は、生成元を与える部分と、それで全てであることを示す部分に分ける。
- 位数計算では軌道安定化定理を使う場合、その前提を明示する。

## 禁止事項・注意事項

- 検証されていない命題を「定理」として置かない。
- 記法 `D_8` など、文献により意味が揺れるものは必ず位数や表示を併記する。
- 外部の計算機実験を使った場合は、その計算が人間の証明なのか、単なる確認なのかを明記する。
